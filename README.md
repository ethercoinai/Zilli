Hermes-NG 是一个面向 AI 自主开发的下一代 Agent 工具工程方案，核心理念是“AI 写 AI”、“评估即开发”、“从环境中来”以及“从 Agent 到 RL”。整个系统围绕一个五阶段执行蓝图构建，形成自我进化的闭环。

**Phase 1：任务定义与评估体系**  
定义可自动验证的任务集（如记忆注入、Skill 边界定位、自我修正等），并设计类型安全的 Agent API 契约（基于 Pydantic）。

**Phase 2：轨迹数据构建与环境模拟**  
搭建模拟沙箱环境，让 Agent 在试错中生成轨迹数据，并建立分层经验回放池（黄金轨迹 + 失败反思轨迹），同时引入轨迹净化机制提升数据质量。

**Phase 3：高效 RL 训练基础设施**  
部署兼容 SGLang（推理）与 Megatron-LM（训练）的异构算力体系，实现长度自适应控制器（Earl 机制）、异步 Rollout 调度器（Windowed FIFO），以及全流程训练入口。

**Phase 4：模型训练与强化学习策略**  
采用 CISPO 算法（带 Clipping 和 KL 惩罚）确保多轮 Agent 训练的稳定性，支持基于规则的 RLVR 奖励塑形，并提供 GRPO/DAPO 基线（无需 Critic Network）。

**Phase 5：自动化进化**  
通过离线进化引擎（DSPy + GEPA）优化生成的 Skill，自动化调度定期进化流程，并实现持续学习机制（吸收生产环境交互数据，触发在线更新）。

Hermes-NG 本质上是一个能“自己构建自己”的生态系统：人类仅定义任务与环境，AI 自主编写、迭代代码与 Skill；系统好坏由可验证任务集客观度量；训练与生产数据反哺，实现持续智能进化。
